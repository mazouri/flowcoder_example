## 1. 产品概述

### 1.1 一句话描述

灵薯智作是一款面向小红书内容创作者和运营团队的 **桌面端专业工具**，提供"采集→拆解→创作→发布→数据分析"的全链路自动化运营能力。

### 1.2 目标用户

| 用户角色 | 典型需求 |
|---------|---------|
| 个人创作者 | 快速找到爆款灵感、AI 辅助写文案、一键发布 |
| MCN 机构 / 运营团队 | 多账号矩阵管理、批量采集素材、数据看板 |
| 电商带货博主 | 爆款拆解、仿写高转化文案、定时发布 |
| 品牌方市场部 | 竞品监控、博主分析、内容批量生产 |

### 1.3 平台支持

- Windows 10 及以上（64 位）
- macOS 10.15 及以上（Intel / Apple M1/M2/M3/M4 芯片）

### 1.4 核心价值主张

- 覆盖小红书运营全流程，从选题到发布一站式完成
- 本地优先 (Local-First) 数据架构，用户数据不上传服务器
- 非侵入式 RPA 仿真技术，模拟真人操作，降低封号风险
- 集成主流大模型 API（火山引擎豆包、阿里百炼通义千问）

## 2. 核心功能模块拆解

### 2.1 一键采集素材

**功能描述：** 自动从小红书网页端抓取笔记内容，包括图片、视频、文案、互动数据等，自动去水印，毫秒级抓取。

| 子功能 | 说明 |
|--------|------|
| 一键采集笔记 | 输入笔记链接或关键词，批量抓取笔记内容 |
| 一键采集博主信息 | 抓取博主主页资料、粉丝数、互动数据 |
| 一键采集博主笔记 | 批量抓取指定博主的所有/部分笔记 |
| 自动去水印 | 抓取时自动去除图片/视频水印 |
| 素材库管理 | 采集的素材统一存入本地素材库，支持分类、搜索 |

**交互流程：**
```
用户输入关键词/链接 → 系统自动在内置浏览器中加载页面
→ Extension 插件解析页面 DOM → 提取图片/文案/数据
→ 自动去水印处理 → 存入本地数据库 → 表格化展示
```

### 2.2 AI 深度拆解

**功能描述：** 输入爆款笔记链接，AI 透视笔记的内容结构、情绪钩子与底层逻辑，还原爆款公式。

| 分析维度 | 说明 |
|----------|------|
| 标题拆解 | 分析标题的结构模式（数字法、疑问法、痛点法等） |
| 内容结构 | 拆解正文段落的逻辑框架（开头钩子、中间价值、结尾 CTA） |
| 情绪分析 | 识别文案中的情绪触发点和共鸣元素 |
| 选题分析 | 归纳爆款选题方向和切入角度 |
| 爆款因子 | 综合分析为什么这篇笔记能成为爆款 |

**交互流程：**
```
用户粘贴爆款笔记链接 → 系统采集笔记全部内容
→ 调用大模型 API 进行结构化分析
→ 以可视化卡片形式展示分析结果
→ 可一键生成仿写文案
```

### 2.3 AI 智能仿写

**功能描述：** 基于拆解结果，AI 自动生成高质量仿写文案，保留爆款基因但内容完全原创。

- 保留原文的结构模式和情绪节奏
- 替换具体内容为用户指定的主题/产品
- 支持多次迭代和风格调整

### 2.4 AI 辅助原创

**功能描述：** 从零开始创作原创内容，AI 提供选题建议、大纲生成、正文撰写等辅助。

### 2.5 AI 灵感裂变

**功能描述：** 基于一个核心主题或爆款笔记，AI 裂变出多个不同角度的内容方向，实现"一变多"的矩阵内容生产。

### 2.6 博主分析

**功能描述：** 深度拆解头部博主，分析其内容策略、发布规律、爆款比例等，复制爆款基因。

| 分析维度 | 说明 |
|----------|------|
| 账号概览 | 粉丝数、获赞数、笔记数等基础数据 |
| 内容分析 | 笔记类型分布、爆款率、更新频率 |
| 发布规律 | 发布时间段、频率分析 |
| 互动分析 | 点赞/收藏/评论比例和趋势 |

### 2.7 矩阵发布

**功能描述：** 多账号一键分发，支持扫码登录和定时自动发布。

| 子功能 | 说明 |
|--------|------|
| 多账号管理 | 管理多个小红书账号，支持扫码登录 |
| 一键发布 | 编辑好的笔记一键发布到指定账号 |
| 定时发布 | 设置发布时间，系统自动在指定时间发布 |
| 批量分发 | 同一内容或不同内容分发到多个账号 |
| 电脑端自动发布 | 无需手机操作，电脑端直接完成发布 |

**交互流程：**
```
创作/编辑笔记 → 选择发布账号（支持多选）
→ 设置发布时间（立即/定时） → 确认发布
→ 系统通过 RPA 在内置浏览器中模拟操作完成发布
→ 发布状态实时反馈
```

### 2.8 全域矩阵智能看板

**功能描述：** 整合采集、创作和发布全流程数据，AI 智能分析，全局掌控流量趋势。

| 看板维度 | 说明 |
|----------|------|
| 总览看板 | 全账号汇总数据，核心指标一目了然 |
| 数据看板 | 各账号的详细运营数据、趋势图 |
| 内容效果分析 | 各笔记的互动数据、爆款概率分析 |
| AI 智能分析 | AI 自动生成数据洞察和优化建议 |

## 6. 关键技术实现详解

### 6.1 内容采集引擎

#### 核心原理

灵薯智作通过 Electron 内嵌 Chromium 浏览器加载小红书网页版，再通过 Extension 插件注入脚本来解析页面内容。

#### 技术细节

```
┌─ Electron 主进程 ─────────────────────────────────┐
│                                                   │
│  1. 创建 BrowserView / BrowserWindow              │
│  2. 加载小红书网页 (https://www.xiaohongshu.com)    │
│  3. 注入 Extension 插件                            │
│                                                   │
│  ┌─ Extension 插件 (Content Script) ────────────┐ │
│  │                                              │ │
│  │  4. 监听页面 DOM 变化 (MutationObserver)       │ │
│  │  5. 解析笔记卡片元素，提取：                    │ │
│  │     - 标题、正文、标签                         │ │
│  │     - 图片 URL (原图地址)                      │ │
│  │     - 视频 URL                               │ │
│  │     - 互动数据 (点赞/收藏/评论)                │ │
│  │     - 博主信息                               │ │
│  │  6. 通过 Chrome Extension API → IPC          │ │
│  │     将数据传回主进程                           │ │
│  │                                              │ │
│  └──────────────────────────────────────────────┘ │
│                                                   │
│  7. 主进程接收数据，去水印处理                       │
│  8. 写入本地 SQLite 数据库                         │
│  9. 通知渲染进程更新 UI                            │
│                                                   │
└───────────────────────────────────────────────────┘
```

#### 去水印技术方案

- **图片去水印：** 获取小红书 CDN 原图地址参数（去除水印参数），或通过图像处理算法（OpenCV / Canvas API）进行水印区域检测与修复
- **视频去水印：** 解析视频源地址获取无水印版本，或通过 FFmpeg 进行视频帧处理

### 6.2 RPA 仿真引擎

#### 核心原理

采用"非侵入式 RPA 仿真技术"，模拟真人的鼠标移动、点击、滚动、键盘输入等行为，而非直接调用 API 接口。

#### 技术实现

| 仿真维度 | 实现方式 |
|----------|---------|
| 鼠标轨迹 | 贝塞尔曲线模拟自然移动路径，添加随机抖动 |
| 点击行为 | 随机化点击位置偏移、按下/松开时间间隔 |
| 滚动行为 | 模拟自然滚动速度和停顿，非匀速 |
| 键盘输入 | 逐字符输入，随机化键入间隔（50-200ms） |
| 操作间隔 | 任务间添加随机延迟（几秒到几十秒），避免机械式高频 |
| 页面浏览 | 模拟真人浏览行为：停留、滚动、回看 |

#### 实现技术

- **Electron WebContents API：** `webContents.sendInputEvent()` 发送鼠标和键盘事件
- **Chrome DevTools Protocol (CDP)：** 通过 `Input.dispatchMouseEvent`、`Input.dispatchKeyEvent` 精确控制输入
- **Puppeteer / Playwright 集成：** 可选方案，通过 CDP 连接到 Electron 内嵌浏览器

### 6.5 多账号管理

#### 技术方案

```
┌───────────────────────────────────────┐
│          账号管理模块                   │
│                                       │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ │
│  │ 账号 A   │ │ 账号 B   │ │ 账号 C   │ │
│  │ Session  │ │ Session  │ │ Session  │ │
│  │ Cookie   │ │ Cookie   │ │ Cookie   │ │
│  │ Profile  │ │ Profile  │ │ Profile  │ │
│  └────┬────┘ └────┬────┘ └────┬────┘ │
│       │          │          │        │
│       ▼          ▼          ▼        │
│  每个账号独立的 Electron Session      │
│  (partition / BrowserSession)        │
│  隔离 Cookie、Cache、Storage         │
└───────────────────────────────────────┘
```

- 使用 Electron 的 `session.fromPartition()` 为每个账号创建独立会话
- 各账号的 Cookie / LocalStorage 完全隔离
- 扫码登录：在内置浏览器中展示小红书登录二维码，用户手机扫码完成登录

### 6.6 自动更新机制

- 使用 `electron-updater` 实现自动检查和下载更新
- 公测阶段高频更新，增量更新减少下载量
- 可能使用差分更新 (delta update) 优化更新体验

## 7. 若从零开发的技术选型建议

### 7.1 桌面应用框架

| 方案 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Electron** | 生态成熟、Web 技术栈、内置 Chromium 完美支持 Extension/RPA | 内存占用大、包体积大 | ⭐⭐⭐⭐⭐ |
| Tauri 2.0 | 包体积小、性能好、Rust 安全性 | 内嵌浏览器控制能力弱、Extension 支持差 | ⭐⭐ |
| CEF (C++) | 性能极致、浏览器控制能力强 | 开发成本高、跨平台复杂 | ⭐⭐⭐ |

**推荐：Electron**，因为此类产品的核心是内嵌浏览器 + Extension 注入 + RPA 自动化，Electron 基于 Chromium 天然适合。

### 7.2 前端技术栈

| 组件 | 推荐方案 | 说明 |
|------|---------|------|
| UI 框架 | React 18/19 + TypeScript | 成熟生态，组件丰富 |
| 样式方案 | Tailwind CSS | 快速开发，原子化 CSS |
| 状态管理 | Zustand 或 Jotai | 轻量、适合 Electron 场景 |
| 路由 | React Router 或 TanStack Router | 桌面端多视图管理 |
| 构建工具 | Vite + electron-vite | 极速 HMR，Electron 集成好 |
| UI 组件库 | Ant Design / Radix UI | 数据表格等复杂组件 |
| 图表库 | ECharts / Recharts | 数据看板可视化 |
| 图标 | Lucide React | 从源码确认使用了 Lucide 图标 |

### 7.3 Electron + 浏览器自动化

| 组件 | 推荐方案 | 说明 |
|------|---------|------|
| Electron 版本 | 最新稳定版 | 保持 Chromium 版本更新 |
| 内嵌浏览器 | BrowserView / WebContents | 加载目标网站 |
| Extension 注入 | `ses.loadExtension()` | Electron 原生支持加载 Chrome Extension |
| RPA 事件模拟 | CDP (Chrome DevTools Protocol) | `Input.dispatchMouseEvent` 等 |
| 页面解析 | Content Script (Extension) | 注入到目标页面的 JS 脚本 |
| 元素定位 | CSS Selector / XPath | 定位页面元素 |

### 7.4 本地数据存储

| 组件 | 推荐方案 | 说明 |
|------|---------|------|
| 数据库 | better-sqlite3 | Electron 中使用 SQLite 的最佳方案 |
| ORM | Drizzle ORM / Kysely | 类型安全的 SQL 查询构建 |
| 加密 | Node.js crypto + OS Keychain | AES 加密 + keytar 库访问系统密钥链 |
| 文件存储 | electron app.getPath('userData') | 素材文件本地存储 |
| 配置 | electron-store | 简单键值对配置持久化 |

### 7.5 AI 集成

| 组件 | 推荐方案 | 说明 |
|------|---------|------|
| AI SDK | 各平台官方 SDK 或直接 HTTP 调用 | 火山引擎 SDK、阿里百炼 SDK |
| Prompt 管理 | 模板引擎 (Handlebars / 自研) | 管理不同功能场景的 Prompt 模板 |
| 流式输出 | SSE (Server-Sent Events) | 实现打字机效果的实时输出 |
| 备选方案 | OpenAI 兼容接口 | 火山引擎和百炼均兼容 OpenAI 接口格式 |

### 7.6 后端服务（轻量）

| 组件 | 推荐方案 | 说明 |

## 8. 风险与合规考量

### 8.1 平台政策风险

| 风险 | 说明 | 应对策略 |
|------|------|---------|
| 小红书反爬 | 平台可能升级反爬机制，导致采集失效 | RPA 仿真 + 行为随机化 + 高频更新适配 |
| 账号封禁 | 批量操作可能触发风控 | 控制操作频率、模拟真人行为、用户教育 |
| 平台 TOS 违规 | 自动化操作可能违反小红书用户协议 | 法务评估、用户免责声明 |

### 8.2 数据安全风险

| 风险 | 说明 | 应对策略 |
|------|------|---------|
| 本地数据泄露 | 设备被盗或恶意软件窃取 | 本地加密 + 系统级密钥链 |
| API Key 泄露 | 用户 API Key 被截获 | 加密存储、HTTPS 传输 |

### 8.3 法律合规

| 维度 | 说明 |
|------|------|
| 内容版权 | 采集他人内容涉及版权，需提醒用户合规使用 |
| 数据隐私 | 采集博主公开数据一般合规，但需遵守个人信息保护法 |
| AI 生成内容 | AI 仿写需确保不构成抄袭，建议用户二次修改 |

### 8.4 技术维护成本

| 维度 | 说明 |
|------|------|
| 小红书改版适配 | 页面 DOM 结构变化需要及时更新 Extension |
| Electron 安全更新 | Chromium 安全漏洞需要及时升级 |
| 大模型 API 变更 | 第三方 API 接口变更需要适配 |
| 多平台兼容 | macOS / Windows 双平台构建和测试 |
