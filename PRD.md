# PRD：OpenClaw 主动式情报秘书与长期记忆系统

## 1. 文档信息
- 产品名称：ClawMind（暂定）
- 版本：PRD v1.0
- 产品类型：个人/小团队内容与认知增长系统
- 交付形态：OpenClaw Agent 工作流 + Skills 编排 + 外部服务可选接入
- 目标用户：个人IP创作者、研究者、行业从业者

---

## 2. 背景与问题定义

### 2.1 用户痛点
1. 每天需要刷抖音/B站/小红书/资讯站，耗时高且信息噪音大。  
2. 听完看完即流失，无法沉淀成长期知识资产。  
3. 想持续产出公众号/音频/视频，但选题、搜集、整理、写作链路过长。  

### 2.2 产品目标
构建“主动到达”的 AI 秘书：  
- 每天固定时间主动汇报你关心的专题  
- 自动筛选、总结、解释“为什么与你相关”  
- 关键信息沉淀为长期记忆并周期复习  
- 一键生成图文、口播音频、视频脚本（可扩展视频成片）

---

## 3. 产品目标与成功指标（KPI）

### 3.1 核心目标
- 用“主动推送”替代“手动刷平台”
- 用“长期记忆库”替代“临时摘要”
- 用“自动生产”提升内容产出效率

### 3.2 指标
- 每日简报打开率 >= 60%
- 简报“有用”反馈率 >= 70%
- 每周记忆入库条目数（人均）>= 15
- 记忆复习完成率 >= 50%
- 从简报到首稿产出时间下降 >= 70%

---

## 4. 产品范围（Scope）

### 4.1 P0（必须）
1. 兴趣配置（主题/子主题/负向兴趣）
2. 每日定时情报简报（Markdown）
3. 事件去重与来源引用
4. 长期记忆入库与检索
5. 口播音频生成（TTS）
6. 推送到指定渠道（飞书/企微/邮件）

### 4.2 P1（应有）
1. 突发事件即时提醒（阈值触发）
2. 每周认知报告（掌握度/盲区）
3. 公众号文章自动首稿（引用附录）
4. 复习机制（1/3/7/30）

### 4.3 P2（增强）
1. 视频脚本与基础视频草稿生成
2. 多账号/多主题协同
3. A/B 标题与发布效果回流优化

---

## 5. 用户故事（User Stories）
1. 作为创作者，我希望每天 8:00 收到“我关心主题”的最新汇报，不再手动刷平台。  
2. 作为学习者，我希望重要信息自动沉淀到记忆库，并定期复习，逐步成为该领域专家。  
3. 作为内容生产者，我希望一键把简报转成公众号文章和口播音频。  
4. 作为管理者，我希望所有结论可追溯来源，降低信息失真风险。  

---

## 6. 功能需求（详细）

### 6.1 模块A：兴趣配置中心
**功能点**
- 配置：主题、关键词、实体（人/公司/产品）、排除项  
- 配置：每日播报时间、条目数上限、提醒阈值  
- 配置：输出偏好（报告风格、音色、时长）

**业务规则**
- 主题最少1个，建议1-5个  
- 每个主题可配置负向关键词  
- 条目上限默认10条，防止信息过载  

**OpenClaw实现**
- Skill：`profile_manager`
- 存储：`topic_profile` 表
- Agent在每次任务前拉取用户配置注入上下文

### 6.2 模块B：多源采集与事件化处理
**功能点**
- 多源接入（RSS/API/公开网页/授权渠道）
- 清洗、去重、事件聚类
- 可信度打分、时效性打分

**业务规则**
- 同一事件跨源合并成1条事件卡  
- 关键结论须绑定至少1条来源链接  
- 高风险/冲突信息打“争议”标签

**OpenClaw实现**
- Skills：
  - `source_fetcher`（抓取）
  - `content_cleaner`（清洗）
  - `event_cluster`（聚类）
  - `credibility_ranker`（评分）
- 输出：`event_items` + `source_refs`
- 说明：平台数据需遵守官方接口与使用条款

### 6.3 模块C：秘书式每日简报（核心）
**功能点**
- 每天定时生成 Markdown 简报  
- 每条结构：
  1) 发生了什么  
  2) 为什么与你相关  
  3) 来源证据  
  4) 建议动作（跟进/入库/忽略）

**业务规则**
- 报告长度可配置（快读版/标准版/深度版）  
- 默认Top10；按相关度+影响力排序  
- 冲突事件优先提示“不确定性说明”

**OpenClaw实现**
- Skill：`daily_brief_generator`
- 模板：结构化 Prompt + Markdown 模板
- 定时：OpenClaw 自动化循环/计划任务（按版本能力配置）

### 6.4 模块D：长期记忆中枢（差异化）
**功能点**
- 自动推荐“建议入库”条目  
- 支持手动确认/驳回  
- 记忆检索、记忆回放、记忆版本更新  
- 反证机制（旧结论过期或被推翻）

**记忆类型**
- 事实记忆 / 概念记忆 / 观点记忆 / 策略记忆 / 反证记忆

**业务规则**
- 记忆状态：候选 -> 已验证 -> 核心 -> 过期  
- 关键事实建议至少2个独立来源后升“已验证”  
- 每条记忆保留版本和更新时间

**OpenClaw实现**
- Skills：
  - `memory_writer`
  - `memory_retriever`
  - `memory_reviewer`（复习出题与回顾）
- 存储建议（低成本）：Markdown + SQLite + 向量索引（Chroma/pgvector）

### 6.5 模块E：自动内容生产（图文/音频/视频）
#### E1 图文
- 输出：公众号首稿（Markdown）
- 结构：标题候选、导语、正文、引用附录
- Skill：`article_generator`

#### E2 音频
- 输出：mp3/wav + 文本时间戳
- Skill：`tts_renderer`（可用 edge-tts 低成本）
- 后处理：`ffmpeg_processor`

#### E3 视频（增强）
- 输出：视频脚本 + 字幕 + 基础合成稿
- Skills：`video_scriptor` + `video_compose`
- 低成本方案：模板化 FFmpeg 合成（先不做重AIGC）

### 6.6 模块F：推送与提醒
**功能点**
- 每日8:00推送报告和音频
- 重大事件即时提醒（阈值触发）
- 多通道：飞书/企微/邮件/IM

**OpenClaw实现**
- Skill：`notifier`
- 机制：幂等发送 + 重试 + 失败告警

## 7. OpenClaw Agent 与 Skills 架构设计

### 7.1 推荐形态（低成本且稳定）
**1个主Agent + 多Skills分工**（不是多Agent乱跑）
- 主Agent：`Secretary-Orchestrator`
- Skills清单（P0）：
  1. `profile_manager`
  2. `source_fetcher`
  3. `event_cluster`
  4. `credibility_ranker`
  5. `daily_brief_generator`
  6. `memory_writer`
  7. `memory_retriever`
  8. `tts_renderer`
  9. `notifier`

### 7.2 任务流水线
`定时触发 -> 拉取兴趣画像 -> 数据采集 -> 去重聚类 -> 排序 -> 生成简报 -> 记忆入库推荐 -> 音频生成 -> 推送 -> 记录日志`

---

## 8. 数据设计（核心表）

1. `user_profile`：用户基础配置  
2. `topic_profile`：兴趣主题、排除项、推送参数  
3. `raw_source_item`：原始采集内容与元数据  
4. `event_item`：事件聚类结果  
5. `event_source_ref`：事件-来源映射  
6. `daily_report`：日报正文、版本、状态  
7. `memory_node`：长期记忆节点（类型/置信度/状态）  
8. `memory_relation`：记忆关系（因果/演进/冲突）  
9. `review_task`：复习任务与完成状态  
10. `delivery_log`：推送结果与重试记录  

---

## 9. 核心算法规则

### 9.1 事件排序公式（建议）
`Score = 0.35相关度 + 0.25影响力 + 0.2可信度 + 0.1时效性 + 0.1新颖度`

### 9.2 入库推荐规则
- Score > 阈值 且 含来源 -> 推荐入库  
- 关键事实若来源冲突 -> 标注争议，暂不自动升核心

### 9.3 复习机制
- 基于 SM-2 思路（1/3/7/30）
- 错题/遗忘内容自动提高复习频率

---

## 10. 非功能需求（NFR）
- 可用性：核心任务成功率 >= 99%
- 时延：8:00报告在配置窗口内完成推送
- 稳定性：采集失败可降级（仅已接入源继续）
- 安全：API密钥加密存储、最小权限、审计日志
- 可追溯：每条结论可回到来源

---

## 11. 合规与风险控制
1. 严格遵守数据源平台条款（优先官方/授权渠道）  
2. 输出中保留来源引用，不直接搬运受限原文  
3. 对争议事实给出不确定性提示  
4. 敏感内容加风险分类与人工复核开关  

---

## 12. 成本方案（你关心的重点）

### 12.1 低成本组合（推荐）
- LLM：本地模型优先 + 低价API兜底  
- TTS：edge-tts（免费）  
- 视频：FFmpeg模板合成（低成本）  
- 存储：SQLite/PostgreSQL + 本地对象存储  
- 推送：飞书/企微机器人（基本免费）

### 12.2 成本优先级
优先保“情报质量 + 记忆沉淀”，视频放P2，避免早期成本膨胀。

---

## 13. 验收标准（UAT）

### P0验收
- 可配置主题并定时收到简报  
- 简报含来源，且事件无明显重复  
- 可一键生成音频并成功推送  
- 可入库并检索历史记忆  

### P1验收
- 每周自动生成认知报告  
- 可触发即时提醒  
- 可输出公众号首稿并附引用附录  

---

## 14. 实施阶段（不按日历，仅按能力闭环）

- **阶段A（P0闭环）**：情报采集 -> 简报 -> 音频 -> 推送 -> 记忆入库  
- **阶段B（P1闭环）**：复习机制 -> 周报 -> 首稿生成  
- **阶段C（P2增强）**：视频自动化 -> 多主题矩阵 -> 效果回流优化
