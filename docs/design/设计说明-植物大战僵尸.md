# 植物大战僵尸 - 设计说明文档

## 1. 信息架构

### 1.1 页面结构

```
┌─────────────────────────────────────────┐
│  标题区：植物大战僵尸                    │
├─────────────────────────────────────────┤
│  状态栏：阳光数 | 开始/暂停/重试 按钮     │
├─────────────────────────────────────────┤
│  植物选择栏：5 种植物卡片                │
├─────────────────────────────────────────┤
│  游戏画布：5×9 草地网格 + 实体渲染        │
├─────────────────────────────────────────┤
│  操作说明区                              │
└─────────────────────────────────────────┘
```

### 1.2 组件层级

- **App**：根容器，管理 GameEngine 状态与事件
- **StatusBar**：阳光显示、游戏控制按钮
- **SeedBank**：植物选择，展示成本与冷却
- **GameCanvas**：Canvas 渲染、点击交互
- **GameOverlay**：胜负弹层

### 1.3 状态机

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| idle | 未开始 | 开始游戏 |
| playing | 进行中 | 暂停、种植、收集阳光 |
| paused | 已暂停 | 继续 |
| won | 胜利 | 再来一局 |
| lost | 失败 | 再来一局 |

## 2. 交互规范

### 2.1 种植流程

1. 用户点击植物卡片 → 卡片高亮（selected）
2. 用户点击草地格子 → 若阳光足够且无冷却，放置植物并扣费
3. 放置后卡片进入冷却，显示剩余秒数
4. 再次点击已选卡片可取消选择

### 2.2 阳光收集

- 阳光每约 8 秒自然掉落一个
- 点击阳光实体即可收集 +25
- 阳光有轻微脉动动画提示可点击

### 2.3 错误处理与边界条件

- 阳光不足：植物卡片置灰，不可选
- 冷却中：卡片显示冷却倒计时，不可选
- 格子已有植物：不可重复放置
- 游戏未开始/已结束：植物选择栏禁用

### 2.4 暂停与继续

- 进行中点击暂停按钮 → 状态变为 paused，游戏循环停止更新
- 暂停中点击继续 → 恢复 playing

## 3. 动效说明

### 3.1 过渡时长

| 元素 | 时长 | 缓动 |
|------|------|------|
| 按钮 hover | 150ms | ease |
| 卡片选中 | 150ms | ease |
| 焦点环 | 200ms | ease |

### 3.2 阳光动画

- 阳光实体：8 射线旋转 + 半径脉动（sin 驱动）
- 脉动周期约 400ms

### 3.3 Reduced Motion

- 全局通过 `@media (prefers-reduced-motion: reduce)` 将动画与过渡时长设为 0.01ms
- 满足无障碍用户偏好

## 4. 无障碍要求

### 4.1 对比度

- 文字与背景对比度 ≥ 4.5:1（WCAG AA）
- 主按钮使用 primary-500 与白色文字

### 4.2 焦点管理

- 所有可点击元素支持键盘焦点
- 焦点环：2px 外描边，offset 2px
- 画布设置 `tabIndex={0}` 可聚焦

### 4.3 ARIA

- 植物选择栏：`role="toolbar"`，`aria-label="植物选择栏"`
- 状态栏：`role="status"`，`aria-live="polite"`
- 植物按钮：`aria-pressed`、`aria-label` 含名称与成本
- 画布：`aria-label` 说明点击用途
- 胜负弹层：`role="dialog"`，`aria-modal="true"`

### 4.4 触摸目标

- 植物卡片、控制按钮最小 44×44px
