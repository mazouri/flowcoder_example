# 植物大战僵尸前端 - 测试计划

## 1. 测试范围

### 测试目标

- 验证游戏启动、暂停、继续、重试流程的正确性
- 验证 5 种植物放置、成本、冷却、攻击/防御/爆炸逻辑
- 验证阳光系统（初始、自然掉落、向日葵产出、点击收集）
- 验证 4 种僵尸生成、移动、啃食、死亡逻辑
- 验证胜负判定（60 秒存活且无僵尸为胜；僵尸到达左边界为败）
- 验证 UI 组件渲染与无障碍属性

### 测试类型

- [x] 功能测试（手动 + 自动化）
- [x] UI 测试（组件测试）
- [x] 单元测试（GameEngine 核心逻辑）
- [x] E2E 测试（主流程）
- [ ] 性能测试（本期不覆盖）
- [ ] 安全测试（本期不覆盖）

### 不在范围内

- 多关卡、存档、移动端适配
- 音效、复杂动画细节
- 跨浏览器兼容性深度测试（以 Chrome 为主）

## 7. 测试策略

### 7.1 单元测试

- **GameEngine**：状态机、放置植物、收集阳光、胜负判定、樱桃炸弹爆炸
- **constants**：配置值校验

### 7.2 组件测试

- **StatusBar**：阳光显示、按钮根据状态切换
- **SeedBank**：植物卡片禁用/选中/冷却状态
- **GameOverlay**：胜负弹层显示与「再来一局」按钮

### 7.3 E2E 测试

- 游戏启动 → 种植 → 收集阳光 → 暂停/继续
- 胜负弹层与重试流程
