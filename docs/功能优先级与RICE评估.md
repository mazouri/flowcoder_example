# 植物大战僵尸前端 - 功能优先级与 RICE 评估

## 功能优先级评估（RICE 框架）

| 功能 | Reach | Impact | Confidence | Effort | RICE Score |
|------|-------|--------|-------------|--------|------------|
| 游戏画布与网格布局 | 高 | 3 | 1.0 | 2 | **15.0** |
| 阳光系统（掉落+收集） | 高 | 3 | 0.95 | 1.5 | **19.0** |
| 豌豆射手种植与攻击 | 高 | 3 | 0.95 | 3 | **9.5** |
| 向日葵种植与产阳光 | 高 | 3 | 0.9 | 2 | **13.5** |
| 普通僵尸生成与移动 | 高 | 3 | 0.95 | 2 | **14.25** |
| 碰撞检测与战斗逻辑 | 高 | 3 | 0.9 | 3 | **9.0** |
| 胜负判定与界面 | 高 | 3 | 1.0 | 1.5 | **20.0** |
| 坚果墙 | 中 | 2 | 0.9 | 1.5 | **12.0** |
| 樱桃炸弹 | 中 | 2 | 0.85 | 2 | **8.5** |
| 路障僵尸 | 中 | 2 | 0.9 | 1.5 | **12.0** |
| 暂停/继续 | 中 | 2 | 1.0 | 0.5 | **40.0** |
| 波次系统 | 高 | 3 | 0.95 | 1.5 | **19.0** |

**RICE 公式**：Score = (Reach × Impact × Confidence) / Effort

### 评分标准说明

- **Reach**：高 = 所有用户必用，中 = 部分场景使用
- **Impact**：3 = 核心体验，2 = 重要体验，1 = 锦上添花
- **Confidence**：0–1，技术可行性与需求确定性
- **Effort**：人天，开发工作量估算

## 开发顺序建议

```
Phase 1（基础框架）
├── 画布与网格布局
├── 阳光系统
└── 基础 UI（阳光数、关卡）

Phase 2（核心玩法）
├── 豌豆射手
├── 普通僵尸
├── 碰撞与战斗
└── 波次系统

Phase 3（闭环与扩展）
├── 向日葵
├── 胜负判定
├── 坚果墙
├── 樱桃炸弹
└── 路障僵尸

Phase 4（体验优化）
├── 暂停/继续
└── 动画与视觉 polish
```

## 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Canvas 性能不足 | 帧率低 | 考虑 Pixi.js/Phaser 等引擎 |
| 碰撞检测复杂 | 开发延期 | 使用矩形碰撞，简化实现 |
| 资源缺失 | 视觉简陋 | 使用占位图或开源素材 |
