# 植物大战僵尸 Web 版 - 产品需求文档 (PRD)

## 1. 概述

| 项目 | 说明 |
|------|------|
| **版本** | v1.0 |
| **功能分类** | 休闲塔防游戏 |
| **实现方式** | 纯前端实现（HTML5 Canvas / WebGL + JavaScript/TypeScript） |
| **状态** | 草稿 |
| **目标平台** | Web 浏览器（桌面端优先） |

---

## 2. 背景与目标

### 2.1 背景

- **市场需求**：塔防类游戏在 Web 端存在空白，用户希望在浏览器中无需下载即可体验经典塔防玩法
- **技术趋势**：现代浏览器 Canvas/WebGL 能力成熟，足以支撑 2D 塔防游戏的流畅运行
- **用户诉求**：碎片化娱乐、即开即玩、无需安装

### 2.2 目标

| 类型 | 目标 |
|------|------|
| **业务目标** | 打造可复用的 Web 塔防游戏 Demo，验证前端游戏开发可行性 |
| **用户目标** | 在浏览器中体验完整的塔防策略乐趣，种植植物抵御僵尸入侵 |

### 2.3 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|----------|
| 关卡完成率 | ≥60% | 用户通关第一关比例 |
| 平均游戏时长 | ≥5 分钟/次 | 单次会话时长 |
| 帧率稳定性 | ≥30 FPS | 性能监控 |
| 浏览器兼容 | Chrome/Edge/Firefox 最新版 | 兼容性测试 |

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 游戏地图

| 项目 | 说明 |
|------|------|
| **输入** | 用户无直接输入 |
| **处理** | 渲染 5 行 × 9 列草坪网格，每格可放置 1 株植物 |
| **输出** | 可视化游戏战场，僵尸从左向右移动，植物在格内攻击 |

#### 3.1.2 植物系统

| 植物类型 | 阳光消耗 | 功能 | 攻击/防御方式 |
|----------|----------|------|---------------|
| 向日葵 | 50 | 生产阳光 | 每 24 秒产出 25 阳光 |
| 豌豆射手 | 100 | 远程攻击 | 发射豌豆，伤害 20/发 |
| 坚果墙 | 50 | 防御 | 生命值 4000，阻挡僵尸 |
| 樱桃炸弹 | 150 | 范围伤害 | 3×3 范围爆炸，伤害 1800 |
| 寒冰射手 | 175 | 减速+伤害 | 发射寒冰豌豆，伤害 20，减速 50% |

**放置逻辑**：
- 点击植物卡片 → 点击草坪格子完成放置
- 放置前需检查：阳光是否足够、格子是否为空
- 放置后扣除阳光，进入冷却（可选）

#### 3.1.3 僵尸系统

| 僵尸类型 | 生命值 | 速度 | 特性 |
|----------|--------|------|------|
| 普通僵尸 | 200 | 慢 | 无特殊能力 |
| 路障僵尸 | 560 | 慢 | 头戴路障，需先打掉路障(200) |
| 撑杆僵尸 | 200 | 快 | 可跳过第一个植物 |

**生成逻辑**：
- 按波次生成，每波间隔可配置
- 僵尸从屏幕右侧 5 行随机出现
- 到达最左侧（房子）则游戏失败

#### 3.1.4 阳光系统

- **初始阳光**：50
- **获取方式**：向日葵产出、天上掉落（可选）
- **消耗方式**：放置植物时扣除
- **显示**：左上角实时显示当前阳光数

#### 3.1.5 胜负判定

| 条件 | 结果 |
|------|------|
| 所有僵尸被消灭且无新僵尸 | 胜利 |
| 任一僵尸到达最左侧 | 失败 |
| 用户主动暂停/退出 | 暂停/结束 |

### 3.2 功能边界

| 支持 | 不支持 |
|------|--------|
| 单关卡（白天草坪） | 多关卡、黑夜/泳池等场景 |
| 5 种植物 + 4 种僵尸 | 完整原版全部植物/僵尸 |
| 鼠标操作 | 触屏优化（本期不做） |
| 桌面浏览器 | 移动端适配（本期不做） |

### 3.3 边界条件

| 类型 | 说明 |
|------|------|
| **数据限制** | 单格最多 1 株植物；阳光上限 9999 |
| **格式要求** | 资源使用 PNG/Sprite 或 SVG，音效 MP3/OGG |
| **异常处理** | 资源加载失败显示提示；游戏崩溃可重新开始 |

## 4. 非功能需求

### 4.1 性能

- 游戏主循环 30–60 FPS
- 首屏加载 < 5 秒
- 资源按需加载，支持预加载进度条

### 4.2 交互

- 点击反馈明确（高亮、音效）
- 植物/僵尸有基础动画（待机、攻击、死亡）
- 提供「开始游戏」「暂停」「重新开始」按钮

### 4.3 视觉

- 风格参考原版，可简化（像素/卡通风格）
- 清晰的 UI：阳光数、植物栏、暂停按钮

---

## 5. 技术依赖

| 类型 | 说明 |
|------|------|
| **核心技术** | HTML5、CSS3、JavaScript/TypeScript |
