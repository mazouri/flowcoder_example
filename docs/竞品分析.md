## 三、差异化策略

### 1. 定位差异化

- **本产品**：Web 端「轻量复刻」—— 在浏览器中还原核心 PvZ 体验，无需下载
- **竞品**：原版/手游以完整内容、内购为主；Web 小游戏以极简、广告为主

### 2. 核心价值主张

| 维度 | 本产品 | 竞品对比 |
|------|--------|----------|
| 便捷性 | 打开链接即玩 | 优于需下载的原版/手游 |
| 还原度 | 核心玩法还原 | 优于多数 Web 塔防小游戏 |
| 纯净度 | 无广告、无内购 | 优于带广告的 Web 小游戏 |
| 学习价值 | 代码可读、可二次开发 | 面向开发者/学习者 |

### 3. 目标用户聚焦

- **主要**：想在浏览器里玩塔防的休闲玩家
- **次要**：学习前端游戏开发的开发者
- **不主打**：追求完整 PvZ 内容的重度玩家（引导至原版/手游）

## 四、功能借鉴建议

| 借鉴来源 | 可借鉴点 | 本产品实现建议 |
|----------|----------|----------------|
| 原版 PvZ | 植物/僵尸数值、关卡节奏 | 参考平衡性，做简化版 |
| 原版 PvZ | UI 布局、阳光显示 | 左上角阳光、底部植物栏 |
| Web 小游戏 | 加载快、资源轻 | 精灵图压缩、按需加载 |
| Web 小游戏 | 单页架构 | 单 HTML，无路由 |

## 五、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 版权/商标 | 不能使用「植物大战僵尸」正式名称 | 使用「塔防花园」「植物守卫」等替代名 |
| 性能 | 低端设备卡顿 | 降帧、简化粒子、提供画质选项 |
| 竞品同质化 | Web 塔防增多 | 强化代码质量、文档、可扩展性 |
