# 植物大战僵尸 - 技术方案

## 1. 需求概述

Web 版塔防游戏，纯前端实现。5×9 草地网格，5 种植物、4 种僵尸，阳光系统，胜负判定（60 秒存活且清空僵尸为胜；僵尸到达左边界为败）。

## 2. 技术选型

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| Vite + React + TS | 快速构建、类型安全、生态成熟 | - | ✅ 采用 |
| Canvas 2D | 轻量、无额外依赖、易调试 | 复杂动画需手写 | ✅ 采用 |
| Tailwind CSS | 原子化、响应式、主题易配 | 类名较多 | ✅ 采用 |
| Pixi.js/Phaser | 高性能、丰富 API | 包体积大、学习成本 | ❌ 本期不采用 |

## 3. 架构设计

### 3.1 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                        App (React)                       │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────────────┐│
│  │  StatusBar  │ │  SeedBank   │ │ GameCanvas+Overlay   ││
│  │ 阳光/控制   │ │ 植物选择    │ │ Canvas 渲染 + 弹层   ││
│  └──────┬──────┘ └──────┬──────┘ └──────────┬──────────┘│
│         │               │                    │          │
│         └───────────────┼────────────────────┘          │
│                         ▼                               │
│                 ┌───────────────┐                       │
│                 │  GameEngine   │  (纯逻辑，无 UI)       │
│                 │ 状态/实体/规则 │                       │

```

### 3.2 数据模型

- **GameState**: `idle` | `playing` | `paused` | `won` | `lost`
- **PlantEntity / ZombieEntity / SunEntity / PeaEntity**: 坐标、尺寸、血量等
- **PlantConfig / ZombieConfig**: 成本、冷却、攻击力等配置

### 3.3 核心 API（GameEngine）

| 方法 | 描述 |
|------|------|
| `start()` | 开始/重置游戏 |
| `togglePause()` | 暂停/继续 |
| `selectPlant(type)` | 选择植物 |
| `placePlant(pos)` | 放置植物 |
| `collectSun(sunId)` | 收集阳光 |
| `update(deltaTime)` | 游戏主循环更新 |
| `subscribe(listener)` | 订阅状态变化 |

## 4. 实现要点

### 4.1 游戏循环

- **playing**: `requestAnimationFrame` 约 60fps
- **paused**: `setInterval(250ms)` 降低 CPU 占用

### 4.2 事件驱动

- `sunChange`、`stateChange` 通过 `subscribe` 通知 React 更新 UI
- 植物冷却通过 100ms 轮询 `getPlantCooldown` 更新

### 4.3 无障碍

- `prefers-reduced-motion` 下动画时长设为 0.01ms
- 焦点环、ARIA、触摸目标 ≥44px
