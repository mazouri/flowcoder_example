# 植物大战僵尸 Web 版 - 需求分析与用户故事

## 一、需求背景

| 项目 | 说明 |
|------|------|
| **来源** | 产品规划 / 技术验证 |
| **问题** | Web 端缺少即开即玩的塔防游戏，用户需下载客户端才能体验 |
| **影响** | 目标用户：休闲游戏玩家、塔防爱好者；使用场景：碎片时间、浏览器内娱乐 |

---

## 二、用户画像

| 画像 | 特征 | 核心诉求 |
|------|------|----------|
| 休闲玩家 | 非硬核玩家，喜欢简单策略 | 快速上手、无需注册、随时可玩 |
| 怀旧玩家 | 玩过原版 PvZ | 还原经典玩法、浏览器即可体验 |
| 前端学习者 | 开发者/学生 | 学习前端游戏开发、可读的代码结构 |

---

## 三、用户故事

### US-01：游戏启动与准备

**作为** 玩家，  
**我希望** 打开页面即可看到开始界面和游戏说明，  
**以便** 快速了解玩法并开始游戏。

**验收标准：**
```
Given 用户访问游戏页面
When 页面加载完成
Then 显示游戏标题、开始按钮、简要玩法说明
And 点击「开始游戏」后进入游戏主界面
```

---

### US-02：种植植物抵御僵尸

**作为** 玩家，  
**我希望** 通过点击选择植物并放置到草坪格子上，  
**以便** 建立防线抵御僵尸进攻。

**验收标准：**
```
Given 游戏已开始且阳光充足
When 用户点击植物卡片（如豌豆射手）
Then 植物卡片高亮，鼠标变为放置光标
When 用户点击空的草坪格子
Then 该格放置对应植物，扣除阳光，植物开始工作（攻击/产阳光）
Given 阳光不足或格子已有植物
When 用户尝试放置
Then 放置失败，给予提示（可选）
```

---

### US-03：收集阳光维持经济

**作为** 玩家，  
**我希望** 通过种植向日葵或拾取掉落阳光来获得阳光，  
**以便** 有足够资源种植更多植物。

**验收标准：**
```
Given 场上存在向日葵
When 经过约 24 秒（可配置）
Then 向日葵产出 25 阳光，阳光数增加
Given 阳光在场上（若支持掉落）
When 用户点击阳光
Then 阳光被收集，阳光数增加
And 左上角阳光数实时更新
```

---

### US-04：消灭僵尸赢得胜利

**作为** 玩家，  
**我希望** 植物能自动攻击僵尸，僵尸生命值归零后消失，  
**以便** 消灭所有僵尸后通关。

**验收标准：**
```
Given 僵尸进入植物攻击范围
When 植物攻击冷却完成
Then 植物发射子弹/造成伤害
When 僵尸生命值 ≤ 0
Then 僵尸播放死亡动画并移除
Given 场上无僵尸且无待生成僵尸
When 最后一波结束
Then 显示胜利界面，可重新开始
```

---

### US-05：失败与重试

**作为** 玩家，  
**我希望** 当僵尸突破防线时能看到失败提示，并能重新开始，  
**以便** 总结经验再次挑战。

**验收标准：**
```
Given 任一僵尸到达最左侧（房子）
When 碰撞检测触发
Then 游戏暂停，显示「游戏失败」界面
And 提供「重新开始」按钮
When 用户点击「重新开始」
Then 重置游戏状态，回到初始界面或直接开始新局
```

---

### US-06：暂停与继续

**作为** 玩家，  
**我希望** 在游戏过程中能暂停和继续，  
**以便** 应对临时打断。

**验收标准：**
```
Given 游戏进行中
When 用户点击「暂停」按钮
Then 游戏逻辑停止，显示暂停遮罩
When 用户点击「继续」
Then 游戏恢复运行
```

---

## 四、验收标准汇总表

| 用户故事 | Given | When | Then |
|----------|-------|------|------|
| US-01 | 访问页面 | 加载完成 | 显示开始界面，可点击开始 |
| US-02 | 阳光充足、格子为空 | 选植物+点格子 | 放置成功，扣阳光 |
| US-02 | 阳光不足/格子有植物 | 尝试放置 | 放置失败 |
| US-03 | 有向日葵 | 24 秒后 | 产出 25 阳光 |
| US-04 | 僵尸在攻击范围 | 冷却完成 | 造成伤害 |
| US-04 | 僵尸生命≤0 | - | 死亡并移除 |
| US-04 | 无僵尸且波次结束 | - | 胜利界面 |
| US-05 | 僵尸到最左侧 | 碰撞 | 失败界面+重试 |
| US-06 | 游戏进行中 | 点暂停 | 暂停；点继续则恢复 |

## 五、成功指标

| 指标 | 目标 | 衡量方式 |
|------|------|----------|
| 首次点击到首次放置 | < 10 秒 | 用户行为埋点 |
| 第一关通关率 | ≥ 60% | 胜利次数/总开局数 |
| 平均单局时长 | 5–10 分钟 | 会话时长统计 |
| 7 日回访率 | ≥ 20% | 用户 ID 回访（若支持） |
