# UI/UX 原型研发结构化交付物

> 本文档为 UI/UX 原型研发的结构化交付物，供研发工程师实现时参考。  
> 与产品经理交付物（PRD、需求分析）衔接，输出设计规范、可运行原型与设计说明文档。

---

## 一、角色能力概览

### 1.1 核心能力矩阵

| 能力域 | 能力项 | 说明 |
|--------|--------|------|
| **设计规范** | 设计令牌 (Design Tokens) | 颜色、间距、字体、圆角等可复用变量 |
| | 组件规范 | 按钮、输入框、卡片等基础组件定义 |
| | 布局规范 | 栅格、断点、响应式规则 |
| **原型设计** | 低保真原型 | 线框图、信息架构、流程草图 |
| | 高保真原型 | 可交互的视觉原型 |
| | 交互说明 | 状态、动效、反馈规则 |
| **设计文档** | 设计说明 | 设计决策、使用场景、约束条件 |
| | 设计评审 | 与研发、产品的对齐与验收 |

### 1.2 协作接口

| 角色 | 输入 | 输出 |
|------|------|------|
| 产品经理 | PRD、用户故事、优先级 | 原型、设计说明、设计评审反馈 |
| 研发工程师 | 技术约束、实现反馈 | 设计规范、可运行原型、组件规格 |
| 测试工程师 | 用例、边界场景 | 原型状态、交互说明 |

## 二、设计规范模板

### 2.1 设计令牌 (Design Tokens)

```yaml
# design-tokens.yaml 示例结构

colors:
  primary:
    50: "#eff6ff"
    500: "#3b82f6"
    600: "#2563eb"
    700: "#1d4ed8"
  neutral:
    50: "#fafafa"
    100: "#f4f4f5"
    500: "#71717a"
    900: "#18181b"

spacing:
  xs: 4
  sm: 8
  md: 16
  lg: 24
  xl: 32

typography:
  fontFamily:
    sans: "Inter, system-ui, sans-serif"
    mono: "JetBrains Mono, monospace"
  fontSize:
    xs: 12
    sm: 14
    base: 16
    lg: 18
    xl: 20
  fontWeight:
    normal: 400
    medium: 500
    semibold: 600
    bold: 700

borderRadius:
  sm: 4
  md: 8
  lg: 12
  full: 9999

breakpoints:
  sm: 640
  md: 768
  lg: 1024
  xl: 1280
```

### 2.2 组件规范表

| 组件 | 变体 | 状态 | 尺寸 | 说明 |
|------|------|------|------|------|
| Button | primary, secondary, ghost, danger | default, hover, active, disabled | sm, md, lg | 主操作、次要操作、危险操作 |
| Input | text, password, search | default, focus, error, disabled | sm, md, lg | 单行文本输入 |
| Card | default, elevated, outlined | - | - | 内容容器 |
| Modal | default, fullscreen | open, closing | - | 弹窗容器 |
| Table | default, striped | - | - | 数据表格 |

### 2.3 布局规范

| 项目 | 规范 |
|------|------|
| 栅格 | 12 列，gutter 24px |
| 容器最大宽度 | 1280px (xl) |
| 侧边栏宽度 | 240px (可折叠) |
| 顶部导航高度 | 64px |

## 三、可运行原型规范

### 3.1 原型目录结构

```
prototype/
├── index.html          # 入口页面
├── pages/              # 页面级原型
│   ├── dashboard.html
│   ├── list.html
│   └── detail.html
├── components/         # 可复用组件片段
│   ├── header.html
│   ├── sidebar.html
│   └── card.html
├── styles/
│   ├── tokens.css      # 设计令牌
│   ├── base.css        # 基础样式
│   └── components.css  # 组件样式
├── scripts/
│   ├── navigation.js   # 路由/导航
│   └── interactions.js # 交互逻辑
└── assets/             # 图片、图标
    └── icons/
```

### 3.2 原型技术要求

| 要求 | 说明 |
|------|------|
| 技术栈 | 纯 HTML/CSS/JS，无需构建，可直接在浏览器打开 |
| 兼容性 | 现代浏览器 (Chrome, Firefox, Safari, Edge) |
| 响应式 | 支持 sm/md/lg 断点 |
| 可访问性 | 基础 ARIA、键盘导航、对比度 |

### 3.3 原型与设计规范映射

- 原型中的样式必须引用 `tokens.css` 中的变量
- 组件命名与设计规范表一致
- 页面结构对应 PRD 中的功能模块

## 四、设计说明文档模板

### 4.1 文档头信息

```markdown
# [功能/页面名称] 设计说明

| 项目 | 内容 |
|------|------|
| 版本 | v1.0 |
| 更新日期 | YYYY-MM-DD |
| 设计者 | - |
| 关联 PRD | [PRD 链接或编号] |
| 原型链接 | [prototype/pages/xxx.html] |
```

### 4.2 核心章节结构

```markdown
## 1. 设计目标
- 用户目标
- 业务目标
- 设计原则

## 2. 信息架构
- 页面层级
- 导航结构
- 内容分组

## 3. 交互设计
### 3.1 主要流程
[流程图或步骤说明]

### 3.2 状态与反馈
| 状态 | 表现 | 触发条件 |
|------|------|----------|
| 加载中 | 骨架屏/Spinner | 数据请求中 |
| 空状态 | 空状态插画+引导 | 无数据 |
| 错误 | 错误提示+重试 | 请求失败 |

### 3.3 动效说明
| 场景 | 动效 | 时长 |
|------|------|------|
| 页面切换 | 淡入 | 200ms |
| 弹窗出现 | 缩放+淡入 | 150ms |

## 4. 视觉规范
- 主色、辅助色
- 字体层级
- 间距规则

## 5. 响应式策略
| 断点 | 布局变化 |
|------|----------|
| < 640px | 单列、底部导航 |
| 640-1024px | 双列、侧边栏折叠 |
| > 1024px | 三列、完整侧边栏 |

## 6. 可访问性
- 焦点顺序
- 键盘快捷键
- 屏幕阅读器支持

## 7. 待研发确认
- [ ] 技术可行性
- [ ] 性能考量
- [ ] 第三方依赖
```

## 五、设计评审检查清单

### 5.1 设计侧

- [ ] 设计规范与 PRD 需求一致
- [ ] 原型覆盖主要用户流程
- [ ] 状态与边界场景有说明
- [ ] 设计说明文档完整

### 5.2 研发侧

- [ ] 组件可复用性评估
- [ ] 技术实现可行性
- [ ] 性能与可访问性影响
- [ ] 与现有系统集成点

### 5.3 产品侧

- [ ] 功能完整性
- [ ] 用户体验符合预期
- [ ] 优先级与排期匹配
